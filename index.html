The BOLT CSS Framework

Flexible Grid System

Don't need a flex layout? Great, use the default grid system to get you where you need to be. If you do need flex, it's as easy as adding a modifier to the end of the class.

Full Customization

The large majority of the framework is customizable via variables, including boolean activations. Don't feel confined to a framework's nomenclature. Tailor it to fit your workflow.

Completely Modular

BOLT was created to be totally modular, using OOCSS to easily explain the functionality attached to the element at a glance. Use less overrides. Build your creativity.

Variable Configuration

SASS Maps

The configuration of BOLT is comprised of standard variables and SASS maps. This is to allow for customization on a quicker and larger scale.

For example, let's consider the map $type-weights. 

$type-weights: (
  light: 300,
  normal: 400,
  semibold: 600,
  bold: 700
);

Sometimes font families use different numbers to correspond to each weight. So, if you decide to change the primary font halfway through a project, there's a good chance something in the map will change. Even if you change light to thin and its corresponding weight to 200, the class of thin-weight will be created and return font-weight:200 upon compilation.

Variable Class Names

In an effort to create a truly customizable experience, a good portion of the class names, including the grid, are customizable via the _config. 

$grid-row-suffix: 'row';
$grid-flex-suffix: 'flex';
$grid-flush-suffix: 'flush';

Line Heights

Each part of the hierarchy includes its own line-height to maintain the golden rule of spacing (in most cases). However, you can apply the class modifier of --long to the end of the class to match the variable for long line-height.

Javascript

There are only a few pieces of JS included with BOLT. This is to keep it largely independent of the project you place it in with no reliance on a specific library.

Modernizr

An expected library for most frameworks, Modernizr allows your project to be seen by more people (that haven't quite caught up yet).

Viewport

Fixes the iOS scaling bug, because, you know, Apple is too busy removing and adding ports to their lineup for dongle dominance.

Responsive Comments

A great library for displaying elements that you need only on certain viewports. This is great for video on desktop, and static images on mobile.
